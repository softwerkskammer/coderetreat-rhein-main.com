What is a Coderetreat?

History

Corey Haines, the initiator of the code retreat concept, used a nice metaphor to explain the concept of this thing called a ‘Coderetreat’. Developing software is a bit like playing an instrument. The big difference is that as a musician you take a lot of time practicing and only a fraction of your time performing on stage. As a developer at work, we are constantly on stage and we never take enough time to practice. In our day to day jobs we have the preassure to get things dune, causing us to rush. And so we try to cut corners, but sometimes we do not cut the corners in the best way because we never have time to explore some alternatives.

Photo by David Giard

The Goal

The goal of a code retreat is to shrink the gap between your current level of your coding skills is and where you want them to be, by practicing the fundamentals of software design. We invite you to write the best code you can without the pressure to get things done. During the retreat we do pair-programming, which is wonderful because you can learn a lot from each other. We write our code using TDD and complying to the rules of simple design. We focus purely on writing great code, there is no pressure to actually produce a working solution for the problem.

The Format

The code retreat is held on a weekend day to enable everyone to participate without having to ask for permission from their employer, just from their partner. The format of a code retreat is as follows:

  • Implement Conway’s Game of Life
  • In each of the 45 minutes sessions write the best code you can with your pair
  • After each session delete your code (We'll tell you why, if you show up!)
  • Ten minutes of retrospective
  • Five minutes break
  • Change your pair
  • Repeat
We have 3 sessions in the morning, after that we have lunch for 1h - 1h ½, and 3 more sessions in the afternoon. At the end we hold a big retrospective. The code retreat is programming language agnostic, you can use any language if you find somebody to pair with.
